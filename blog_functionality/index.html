Version 0.4.0 Introduces Blog Functionality

Wiki style blogging

Wikismith users blog by adding page snippets to an index page. No mroe second guessing how the engine will find, index, sort and categorize their posts. You no longer have to hack the source if you want to slightly customize the order - or keep a few posts featured. Just put the posts on the index in the order you want. Don't like a post so much, roll it off the front page early.

Page Snippets

Page snippets are tripple bracketed links called that wikismith converts into pages and replaces with page snippets.

[[[Most Recent Entry]]]

[[[Second Blog Entry]]]

[[[First Blog Entry]]]

Wikismith uses formatters to style the following information the page snippet.

  • Title

  • Created Date

  • Author

  • Excerpt

Similarly to double brackets, Wikismith will create the new page if it doesn't already exist.

Formatters

Wikismith formats page snippets via a newly introduced concept of formatters, and plans to allow these to be overrided by themes in version 0.5.0.

Below is an example formatter for page_snippets.

formatters/index.js

var ejs = require('ejs');
var fs = require('fs');
var path = require('path');

function page_snippet(file) {
    var template = String(fs.readFileSync(path.join(__dirname, 'page_snippet.html')));

    return ejs.render(template, {
        url: file.url,
        params: file.params
    });
}

module.exports = {
    page_snippet: page_snippet
}

formatters/page_snippet.html

<div class="blog-post">
    <h3 class="blog-post-title"><%=params.title%></h3>
    <p class="blog-post-meta"><%=params.created%> by <%=params.author%></p>
    <p><%=params.excerpt%></p>
    <p><a href="<%=url%>">Read More</a></p>
</div>
